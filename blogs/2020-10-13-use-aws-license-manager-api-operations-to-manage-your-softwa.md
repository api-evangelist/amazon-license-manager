<p>The simplified Bring Your Own License (BYOL) experience in&nbsp;<a href="https://aws.amazon.com/license-manager/">AWS License Manager</a> allows you to effectively manage your own software licenses in AWS. These licenses can be used with <a href="https://aws.amazon.com/ec2/">Amazon&nbsp;Elastic Compute Cloud (Amazon EC2)</a> shared tenancy, <a href="https://aws.amazon.com/ec2/dedicated-hosts/">EC2 Dedicated Hosts</a>, <a href="https://aws.amazon.com/rds/">Amazon RDS</a>, <a href="https://aws.amazon.com/marketplace/">AWS Marketplace</a>, or an on-premises environment.&nbsp;License Manager helps you manage your software licenses, including Microsoft Windows Server and Microsoft SQL Server licenses. License administrators can automate license management by building custom dashboards and track licenses centrally using License Manager API operations.</p> 
<h3>Solution tutorial</h3> 
<p>In this blog post, I walk you through the following scenarios.</p> 
<ul> 
 <li><strong>Scenario 1:</strong> Using License Manager API operations to track your license usage across Amazon EC2.</li> 
 <li><strong>Scenario 2:</strong> Using License Manager API operations to manage BYOL Oracle licenses for databases running on Amazon RDS for Oracle.</li> 
 <li><strong>Scenario 3:</strong> Using License Manager API operations to track Windows Server and SQL Server licenses on EC2 Dedicated Hosts.</li> 
 <li><strong>Scenario 4:</strong> Using License Manager API operations to track your license usage across on-premises servers.</li> 
</ul> 
<h3><strong>Scenario 1:&nbsp;Using License Manager API operations to track your license usage across Amazon EC2.</strong></h3> 
<p>If you have Amazon EC2 instances consuming software licenses before you set up the <a href="https://docs.aws.amazon.com/license-manager/latest/userguide/license-configurations.html">license configuration</a>, make sure that these instances are managed by <a href="https://aws.amazon.com/systems-manager/">AWS Systems Manager.</a></p> 
<ul> 
 <li>Follow the steps in <a href="https://docs.aws.amazon.com/systems-manager/latest/userguide/systems-manager-setting-up.html">setting up AWS Systems Manager</a> in the AWS Systems Manager documentation.</li> 
 <li>Install Systems Manager Agent for your operating system using <a href="https://docs.aws.amazon.com/systems-manager/latest/userguide/getting-started-agent.html">SSM Agent installation guide</a>.</li> 
</ul> 
<p>The instances will be managed by Systems Manager. Associate the license configuration to the discovered instances so License Manager can track the license usage.</p> 
<p>Now for or this scenario, I have the software installed and licensed in an <a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/AMIs.html">Amazon Machine Image (AMI)</a>.</p> 
<h4>Step 1</h4> 
<p>To create the license configuration, use the <code>create-license-configuration</code>API in the <a href="https://docs.aws.amazon.com/cli/latest/index.html">AWS CLI</a>.&nbsp;The license configuration specifies how your licenses should be counted (for example, by vCPUs or number of instances). The license configuration in this scenario specifies vCPUs as the counting model for license usage.&nbsp;For information, check&nbsp;<a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/license-manager/create-license-configuration.html">create-license-configuration</a> in the AWS CLI command reference.</p> 
<p><code>$ aws license-manager create-license-configuration \</code><br /> <code>--name license-configuration-tracking-XYZLicense \</code><br /> <code>--license-counting-type vCPU \</code><br /> <code>--license-count 6</code></p> 
<h4>Step 2</h4> 
<p>Use the <code>update-license-specifications-for-resource</code> API operation to associate the license configuration you created in Step1 to the AMI. Replace the resource name in <code>--resource-arn</code> with the ARN of the existing EC2 instance. For information, check <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/license-manager/update-license-specifications-for-resource.html">update-license-specifications-for-resource</a>&nbsp;in the AWS CLI command reference.</p> 
<p><code>$ aws license-manager update-license-specifications-for-resource \</code><br /> <code>--resource-arn arn:aws:ec2:us-west-2::image/ami-00cXXXXXXXXXXX14f \</code><br /> <code>--add-license-specifications LicenseConfigurationArn=arn:aws:license-manager:us-west-2:90XXXXXXX930:license-configuration:lic-18ba608184961fd50ad6b405ffb6a5b5</code></p> 
<h4>Step 3</h4> 
<p>Now that the license configuration is ready, you can use the <code>run-instances</code> API to launch a new EC2 instance from the AMI, which is configured with the license configuration. For information, check <a href="https://docs.aws.amazon.com/cli/latest/reference/ec2/run-instances.html">run-instance</a>&nbsp;in the AWS CLI command reference.</p> 
<p><code>$ aws ec2 run-instances \</code><br /> <code>--instance-type c5.2xlarge \<br /> --image-id ami-00cXXXXXXXXXXX14f \<br /> </code><code>--subnet-id subnet-0fXXXXXXXXXXXXdcf</code></p> 
<h4>Step 4</h4> 
<p>When instances are launched using the AMI, you can now check how many licenses have been consumed.&nbsp;Use&nbsp;<code>get-license-configuration</code> to get license consumption details.&nbsp;For information, check&nbsp;<a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/license-manager/get-license-configuration.html">get-license-configuration</a> in the AWS CLI command reference.</p> 
<p><code>$ aws license-manager get-license-configuration \</code><br /> <code>--license-configuration-arn arn:aws:license-manager:us-west-2:90XXXXXXX930:license-configuration:lic-18ba608184961fd50ad6b405ffb6a5b5</code></p> 
<p>Comparing the sum of <strong>consumedLicenses</strong> with <strong>licenseCount</strong> field of the output, you can find whether the total number of resources that are actively consuming the software license exceeds the number of available licenses.</p> 
<h3>Scenario 2: Using License Manager API operations to manage BYOL Oracle licenses for databases running on Amazon RDS for Oracle.</h3> 
<h4>Step 1</h4> 
<p>First, use the <code>create-license-configuration</code> API operation to create a license configuration to track the number of&nbsp;Oracle licenses you own for databases running on <a href="https://aws.amazon.com/rds/oracle/">Amazon RDS for Oracle</a>.&nbsp;For information about the supported product types for&nbsp;Oracle database running on Amazon RDS, check <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/license-manager/create-license-configuration.html">create-license-configuration</a> in the AWS CLI command reference.</p> 
<p>Using the License Manager dashboard, you can also manage Oracle licenses for databases running on Amazon RDS across AWS accounts centrally. You create licensing rules in the management account, attach them to resources in member accounts, and track usage of licenses across AWS accounts.</p> 
<p>In this scenario, we choose to track licenses used by Oracle databases running Enterprise Edition on Amazon RDS.</p> 
<p><code>$ aws license-manager create-license-configuration \ </code><br /> <code>--name license-configuration-oracle-byol \</code><br /> <code>--description 'This license configuration tracks the consumption of licenses &nbsp;by Oracle database running Enterprise Edition in Amazon RDS'&nbsp;\</code><br /> <code>--license-counting-type vCPU \</code><br /> <code>--license-count 100&nbsp;\</code><br /> <code>--product-information-list file://product_information.json</code></p> 
<p>where the contents of <strong>product_information.json </strong>is as below,</p> 
<p><code>[{</code><br /> <code>&nbsp;"ResourceType": "RDS",</code><br /> <code>&nbsp;"ProductInformationFilterList": [{</code><br /> <code>&nbsp;"ProductInformationFilterName": "Engine Edition",</code><br /> <code>&nbsp;"ProductInformationFilterValue": ["oracle-ee"],</code><br /> <code>&nbsp;"ProductInformationFilterComparator": "EQUALS"}]}]</code></p> 
<h4>Step 2</h4> 
<p>On an ongoing basis, License Manager discovers which resources are used in your environment. The discovery of these resources is not immediate. It might take few minutes.&nbsp;After this resource usage update, License Manager reports the licenses used. For the purposes of this scenario, I followed the steps in <a href="https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/CHAP_GettingStarted.CreatingConnecting.Oracle.html">creating an Oracle DB instance and connecting to a database on an Oracle DB instance.</a> I set up Oracle Enterprise Edition running on Amazon RDS on t3.<a href="https://aws.amazon.com/ec2/instance-types/">2xlarge</a>, which has 8 vCPUs and 32 GiB of RAM. You can use the AWS CLI <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/license-manager/get-license-configuration.html">get-license-configuration</a> command to get license consumption details. Note that this instance type is not included in the <a href="https://aws.amazon.com/free/">AWS Free Tier</a> so you will incur charges.</p> 
<p>If you want to list all license usage records for the selected license configuration, you can use the&nbsp;<code>list-usage-for-license-configuration</code>&nbsp;API operation. For information, check <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/license-manager/list-usage-for-license-configuration.html">list-usage-for-license-configuration</a>&nbsp;in the AWS CLI command reference.</p> 
<p><code>$ aws license-manager get-license-configuration \</code><br /> <code>--license-configuration-arn arn:aws:license-manager:us-west-2:90XXXXXXX930:license-configuration:lic-86ba1b038d3a794dd8130ddbcf0ed8b3</code></p> 
<h3>Scenario 3: Using License Manager API operations to track Windows Server and SQL Server licenses on EC2 Dedicated Hosts.</h3> 
<p>In this scenario, I show you how the BYOL feature in License Manager simplifies the management of your own software licenses. There are licenses for Windows and SQL Server, that require a dedicated physical server. AWS License Manager is integrated with Amazon EC2 Dedicated Host. In License Manager, you can specify your licensing terms for governing license usage and your Dedicated Host management preferences for host allocation and host capacity utilization. After the setup, AWS takes care of these administrative tasks on your behalf. You can seamlessly launch VMs (instances) on Dedicated Hosts same as how you would launch an EC2 instance with AWS-provided licenses.</p> 
<h4>Step 1</h4> 
<p>Use <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/license-manager/create-license-configuration.html">create-license-configuration</a> to create a license configuration to track the number of licenses you own (in this example, for Windows Server).</p> 
<p><code>$ aws license-manager create-license-configuration \</code><br /> <code>--name lc-win-byol --description 'This license configuration tracks the consumption of my Windows server licenses' \</code><br /> <code>--license-counting-type Core \</code><br /> <code>--license-count 100 \</code><br /> <code>--license-rules #allowedTenancy=EC2-DedicatedHost</code></p> 
<h4>Step 2</h4> 
<p>Next, use the <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/license-manager/update-license-specifications-for-resource.html">update-license-specifications-for-resource</a> API operation to associate your AMI to the launch configuration you just created.</p> 
<p><code>aws license-manager update-license-specifications-for-resource \</code><br /> <code>--resource-arn arn:aws:ec2:us-west-2::image/ami-0eXXXXXXXXXXXXef9 &nbsp;\</code><br /> <code>--add-license-specifications LicenseConfigurationArn=arn:aws:license-manager:us-west-2:90XXXXXXX930:license-configuration:lic-20ba1b8f56073f7f3e4e1c247b96dfda</code></p> 
<h4>Step 3</h4> 
<p>After you have created license configurations and associated them with the AMI, use the AWS License Manager console to create a host resource group and associate it with the license configuration. Because you are using only those licenses on hosts that belong to this group, choose the license configurations you created earlier. For more information, check <a href="https://docs.aws.amazon.com/license-manager/latest/userguide/host-resource-groups.html#host-resource-group-create">create a host resource group</a> in the AWS License Manager documentation. As of the writing of this blog post, you cannot use the API to create a host resource group.</p> 
<p style="text-align: center;"><img alt="Picture showing host resource group in the AWS License Manager console" class="size-large wp-image-13172" height="996" src="https://d2908q01vomqb2.cloudfront.net/972a67c48192728a34979d9a35164c1295401b71/2020/10/13/devops-760_HRG_Console_lc-1-1024x996.jpg" width="1024" /><br /> Figure 1: Host resource groups page of the AWS License Manager console</p> 
<p>After the host resource group is created and configured, you can launch instances using their AMI. AWS places instances on an automatically allocated host inside the <em>HRG-lc-dedicated_capacity</em> host resource group.</p> 
<h4>Step 4</h4> 
<p>Now that you have created a host resource group and associated license configurations, use the <a href="https://docs.aws.amazon.com/cli/latest/reference/ec2/run-instances.html">run-instances</a> API operation to launch a new EC2 instance from the AMI to which the license configurations are attached.</p> 
<p><code>aws ec2 run-instances \</code><br /> <code>--instance-type c5.2xlarge \<br /> --image-id ami-0eXXXXXXXXXXXXef9 &nbsp;\</code><br /> <code>--subnet-id subnet-0fdb525e9a09a6dcf</code></p> 
<p><strong>Note:</strong>&nbsp; If there is only one host resource group mapped to associated host-based (core/socket) licenses, you do not need to reference&nbsp;a host resource group name in the <code>run-instances</code> call. For more information, check the <a href="https://aws.amazon.com/blogs/mt/simplified-byol-experience-using-aws-license-manager/">simplified Bring-Your-Own-License experience using AWS License Manager</a> blog post written by my colleague, Harshitha Putta, where she discusses the automatic scaling of Dedicated Hosts with BYOL licenses.</p> 
<h4>Step 5</h4> 
<p>Now, monitor the license consumption on the EC2 Dedicated Hosts. Use the <code>list-usage-for-license-configuration</code> API to list all license usage records for the selected license configuration and then choose the host ID (prefixed with <em>h-</em>) from the <code>ResourceArn:</code> field. The host ID is the dedicated host-level information on which the resources consuming your licenses are located. For more information, check <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/license-manager/list-usage-for-license-configuration.html">list-usage-for-license-configuration</a> in the AWS CLI command reference. If you have not created the host resource group, the output of the API call is empty. Because I chose the automatic host allocation setting (check Figure 1) when I created the host resource group, a new host is allocated automatically during the launch of an EC2 instance.</p> 
<p><code>aws license-manager list-usage-for-license-configuration \</code><br /> <code>--license-configuration-arn arn:aws:license-manager:us-west-2:90XXXXXXX930:license-configuration:lic-20ba1b8f56073f7f3e4e1c247b96dfda</code></p> 
<p>You can use the <code>describe-hosts</code>&nbsp;API with the host ID as a parameter retrieved from the previous step.&nbsp;For more information, check <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/ec2/describe-hosts.html">describe-hosts</a>&nbsp;in the AWS CLI command reference. This command lists instances that consume capacity, available free capacity, and other useful parameters of the specified Dedicated Host.</p> 
<p><code>aws ec2 describe-hosts --host-ids h-043da20fa7d9f3a63</code></p> 
<p>Depending on your EC2 Dedicated Host management settings in the host resource group,&nbsp;Amazon EC2 releases unused hosts on your behalf. An unused host has no running instances.</p>

<h3>Scenario 4: Using License Manager API operations to track your license usage across on-premises servers.</h3> 
<p>If you have on-premises servers already consuming software licenses before you set up the license configuration, first make sure that those servers are managed by AWS Systems Manager (steps 1–5).</p> 
<h4>Step 1</h4> 
<p>Servers and VMs in the&nbsp;on-premises environment require an IAM role to communicate with Systems Manager. For more information, check&nbsp;<a href="https://docs.aws.amazon.com/systems-manager/latest/userguide/sysman-service-role.html">create an IAM service role for a hybrid environment</a> in the AWS Systems Manager documentation.</p> 
<h4>Step 2</h4> 
<p>To set up servers and VMs in your hybrid environment as managed instances, you must create a managed instance activation.&nbsp;For more information, check <a href="https://docs.aws.amazon.com/systems-manager/latest/userguide/sysman-managed-instance-activation.html#create-managed-instance-activation-commandline">create a managed-instance activation for a hybrid environment</a>&nbsp;in the AWS Systems Manager documentation.</p> 
<h4>Step 3</h4> 
<p>Install SSM Agent in your hybrid environment so your servers and VMs can&nbsp;communicate with Systems Manager. If you are&nbsp;installing SSM Agent on Linux machines in a hybrid environment, follow the <a href="https://docs.aws.amazon.com/systems-manager/latest/userguide/sysman-install-managed-linux.html">documentation for SSM Agent installation for Linux OS</a>. If you are&nbsp;installing SSM Agent on Windows machines in a hybrid environment,&nbsp;follow <a href="https://docs.aws.amazon.com/systems-manager/latest/userguide/sysman-install-managed-win.html">documentation for SSM Agent installation for Windows OS</a>.</p> 
<h4>Step 4</h4> 
<p>The server or VM in the hybrid environment is now a managed instance and its name is prefixed with <em>mi-</em>. Use the AWS CLI <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/ssm/describe-instance-information.html">describe-instance-information</a> command to view managed instances.</p> 
<p><code>$ aws ssm describe-instance-information</code></p> 
<h4>Step 5</h4> 
<p>Configure the Systems Manager inventory to collect metadata from your managed instances. For more information, check&nbsp;<a href="https://docs.aws.amazon.com/systems-manager/latest/userguide/sysman-inventory-cliwalk.html#sysman-inventory-cliwalk-all">quickly configure your managed instances for inventory by using the AWS CLI&nbsp;</a>in the AWS Systems Manager documentation.</p> 
<p><code>$ aws ssm create-association --name AWS-GatherSoftwareInventory \<br /> </code><code>--targets Key=InstanceIds,Values=mi-0b482706db2c0e4eb \<br /> </code><code>--schedule-expression "rate(1 day)" \<br /> </code><code>--parameters applications=Enabled, awsComponents="Enabled,customInventory=Enabled, \<br /> instanceDetailedInformation=Enabled, networkConfig=Enabled, \<br /> services=Enabled, windowsRoles=Enabled, windowsUpdates=Enabled"</code></p> 
<h4>Step 6</h4> 
<p>Associate the license configuration to the managed instances so that License Manager can track the license usage in the hybrid environment. Create the license configuration before you attempt this step.</p> 
<p><code>$ aws license-manager update-license-specifications-for-resource \</code><br /> <code>--resource-arn arn:aws:ssm:us-east-1:90XXXXXXX930:managed-instance/mi-0b482706db2c0e4eb \</code><br /> <code>--add-license-specifications LicenseConfigurationArn=arn:aws:license-manager:us-east-1:90XXXXXXX930:license-configuration:lic-b8b9621d7683cc59925c0f4aaf80d37d</code></p> 
<h4>Step 7</h4> 
<p>Use the&nbsp;<code>get-license-configuration</code> command to get license consumption details.&nbsp;For information, check&nbsp;get-license-configuration&nbsp;in the AWS CLI command reference.</p> 
<p><code>$ aws license-manager get-license-configuration \</code><br /> <code>--license-configuration-arn&nbsp;arn:aws:license-manager:us-east-1:90XXXXXXX930:license-configuration:lic-b8b9621d7683cc59925c0f4aaf80d37d</code></p> 
<p>Comparing the sum of <strong>consumedLicenses</strong> with <strong>licenseCount</strong> field of the output, you can find whether the total number of resources that are actively consuming the software license exceeds the number of available licenses.</p> 
<h3>Cleaning up</h3> 
<p>If you followed along and created AWS resources, we recommend that you clean up the environment if it is not required. You will incur charges for the Amazon RDS database instance type used in Scenario 2 because it exceeds what’s offered by the <a href="https://aws.amazon.com/free">AWS Free Tier</a>.</p> 
<p>To do this cleanup, delete the Amazon RDS database created in Scenario 2 and then the instance launched in Scenarios 1 and 3. Shut down the active Dedicated Host created during Scenario 3. Finally, clean up the AWS License Manager configuration rules.</p> 
<h3>Conclusion</h3> 
<p>In this post, I showed you how to manage your software licenses on Amazon EC2, on-premises. We also discussed how managing BYOL licenses on Amazon RDS for Oracle, and Windows Server and SQL Server licenses on EC2 Dedicated Hosts has become easier with License Manager API operations.&nbsp;You can use AWS License Manager to manage software licenses from a variety of software vendors such as Microsoft, SAP, and IBM across AWS and on-premises environments. For simplicity, I showed the configuration in a single AWS account, but you can also use AWS License Manager to track software licenses across multiple AWS accounts. For more information, check the <a href="https://aws.amazon.com/blogs/mt/tracking-software-usage-across-multiple-aws-accounts-using-aws-license-manager/">tracking software usage across multiple AWS accounts using AWS License Manager</a> blog post. I encourage you to use what you have learned here to reduce the effort of managing licenses across your organization. You might start by using AWS License Manager in a small part of the organization to demonstrate a proof of concept to your peers.</p>
