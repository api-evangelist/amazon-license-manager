<p>Many enterprises use <a href="https://www.google.com/url?sa=t&amp;rct=j&amp;q=&amp;esrc=s&amp;source=web&amp;cd=&amp;cad=rja&amp;uact=8&amp;ved=2ahUKEwiRpsuN3u72AhUqJTQIHaMBCHYQFnoECAwQAQ&amp;url=https%3A%2F%2Faws.amazon.com%2Forganizations%2F&amp;usg=AOvVaw2xxwEjN2738r0SHh7-gUVV">AWS Organizations</a> to link their AWS accounts and manage them using a single management account. Earlier, customers managed license administration using the management account. However, the management account has a number of administrative privileges for the organization, and it’s a best practice to reduce the amount of users in the management account where possible. Then, <a href="https://aws.amazon.com/license-manager/">AWS License Manager</a> announced support for Delegated Administrator, a feature that allows license administrators to manage and distribute licenses across all of their AWS accounts from a delegated member account in the organization.&nbsp;Refer <a href="https://aws.amazon.com/about-aws/whats-new/2021/08/aws-license-manager-supports-delegated-administrator/">AWS License Manager now supports Delegated Administrator</a> announcement to learn more. This helped organizations achieve the flexibility of reducing users in the management account, and administering license management activities using the delegated member account.</p> 
<p>Once the delegated administrator account is configured, you can perform central license administration tasks, such as managing licenses and <a href="https://aws.amazon.com/ec2/">Amazon Elastic Compute Cloud (Amazon EC2)</a> Dedicated Hosts, sharing license configuration with other accounts, and setting up cross-account discovery.</p> 
<p>In this post, you’ll learn how to create a basic organization set up, with one management and two member accounts. Then, you’ll learn how to register&nbsp;the delegated administrator and perform central license administration tasks in the organization, such as setting up cross-account discovery, sharing a simple license configuration with other member accounts, and managing licenses. Finally, you’ll learn how to set up an&nbsp;<a href="https://aws.amazon.com/iam/">AWS Identity and Access Management (IAM)</a>&nbsp;user in the Delegated Administrator account&nbsp;to manage licenses from different license vendors.&nbsp;Creating IAM user(s) under the Delegated Administrator account will help you assign different license management tasks for the specific vendor(s).</p> 
<p>Let’s get started.</p> 
<h2>Prerequisites</h2> 
<p>You’ll need the following prerequisites to implement the solution discussed in this post:</p> 
<ol> 
 <li>An organization with at least three accounts. Refer the following<strong> Setting up your organization</strong>&nbsp;section for instructions if you’re looking to set up an organization and create or invite member accounts into the organization.</li> 
</ol> 
<h2>&nbsp; &nbsp; &nbsp; Setting up your organization</h2> 
<ol> 
 <li> 
  <ol type="a"> 
   <li>Create your organization by selecting an account that doesn’t run existing workloads. This will be the management account for your organization. Refer to the<a href="https://docs.aws.amazon.com/organizations/latest/userguide/orgs_manage_org_create.html#create-org"> Create an organization</a> documentation to learn about how to create an organization by using the AWS Management Console. Once you’ve created the organization, follow Step b to create or invite two other accounts as member accounts into the organization.</li> 
   <li>For this post, invite accounts to join the organization. Refer to the <a href="https://docs.aws.amazon.com/organizations/latest/userguide/orgs_manage_accounts_invites.html#orgs_manage_accounts_invite-account">Sending invitations to AWS accounts</a> documentation to learn about how to create or invite an account to be part of your organization.</li> 
   <li>The invited account should accept the invitation to join the organization as a member account. Refer to the <a href="https://docs.aws.amazon.com/organizations/latest/userguide/orgs_manage_accounts_invites.html#orgs_manage_accounts_accept-decline-invite">Accepting or declining an invitation from an organization</a> documentation to learn about how to accept or decline the invitation from an organization.</li> 
   <li>If you choose to create a member account to be part of the organization, then AWS automatically creates a role in the account that grants administrator permissions to IAM users in the management account who can assume the role. Refer to the&nbsp;<a href="https://docs.aws.amazon.com/organizations/latest/userguide/orgs_manage_accounts_access.html#orgs_manage_accounts_access-cross-account-role">Accessing a member account that has a management account access role</a> documentation to learn about how to grant permissions to user(s) in the management account to access the role.<br /> Note that member accounts that you invite to join your organization <strong>do not</strong> automatically get an administrator role created. You must do this manually, and this essentially duplicates the role automatically set up for created accounts. Refer to the&nbsp;<a href="https://docs.aws.amazon.com/organizations/latest/userguide/orgs_manage_accounts_access.html#orgs_manage_accounts_create-cross-account-role">Creating the OrganizationAccountAccessRole in an invited member account</a> documentation to learn about how to create a role in a member account that grants administrator permissions to IAM users in the management account who can assume the role.</li> 
   <li>Repeat Steps b – d for the remaining member account.</li> 
  </ol> </li> 
</ol> 
<ol start="2"> 
 <li>A software license bought from a vendor for which vCPU is the counting model. The other supported values are physical core, socket, and instance.</li> 
</ol> 
<h2>Solution walkthrough</h2> 
<h2>Enabling the Delegated Administrator feature</h2> 
<ol> 
 <li>From the <a href="https://us-east-1.console.aws.amazon.com/license-manager/home?region=us-east-1#/">License Manager console</a> of the organization’s management account, enable the option to distribute managed entitlements or license configurations with your organization. Refer to the <a href="https://docs.aws.amazon.com/license-manager/latest/userguide/settings.html">Settings in License Manager</a> documentation to learn how to enable the options to distribute managed entitlements or license configurations with your organization.<br /> As an alternate, once you set up the organization, you can choose not to enable the option to distribute managed entitlements or license configurations with your organization in the management account and still enable a delegated administrator. Specifically, service access for the License Manager service must be enabled in the organization settings to enable the delegated administrator. This way of enabling the delegated administrator doesn’t create a resource share in the management account, since the cross-account resource discovery mode is off.</li> 
 <li>From the License Manager console of the organization’s management account, refer to the <a href="https://docs.aws.amazon.com/organizations/latest/userguide/services-that-can-integrate-license-manager.html#integrate-enable-da-license-manager">Enabling a delegated administrator account for License Manager</a> documentation to learn how to register a delegated administrator account for License Manager. Additionally, you may call the <a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/organizations/list-delegated-administrators.html">list-delegated-administrator</a>&nbsp;AWS Command Line Interface (AWS CLI) command to verify whether or not the specified member account has been successfully registered as a delegated administrator.</li> 
 <li>From the License Manager console of the delegated administrator account, enable the cross-account resource discovery setting in the delegated administrator account. Refer to the <a href="https://docs.aws.amazon.com/license-manager/latest/userguide/settings.html">Settings in License Manager</a> documentation to learn how to enable cross-account resource discovery in the delegated administrator account. This will create a new <a href="https://aws.amazon.com/ram/">AWS Resource Access Manager (AWS RAM)</a> Resource share to share license configurations with the organization and <a href="http://aws.amazon.com/s3">Amazon Simple Storage Service (Amazon S3)</a> creates an S3 bucket in the delegated administrator account to which all other accounts will sync their data. The setting also enables the option to distribute managed entitlements or license configurations with your organization.</li> 
</ol> 
<p style="padding-left: 40px;">Choose your management account to be in single account mode and have the delegated&nbsp;administrator account in the cross-account mode. This means that you’re decoupling license management responsibilities to the&nbsp;delegated&nbsp;administrator account.</p> 
<h2>Delegate your license management on AWS</h2> 
<ol> 
 <li>Procure a software license and bake into your <a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/AMIs.html">Amazon Machine Image (AMI)</a>. Make sure that the AMI has permissions enabled to be shared with your organization. Refer to the<a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/share-amis-with-organizations-and-OUs.html#share-amis-org-ou-console"> Share an AMI (console)</a> documentation to learn about how to share an AMI with an organization.</li> 
 <li>From the License Manager console of the&nbsp;delegated administrator account, create a license configuration and associate it to the AMI. From the license configuration console, you can see how many licenses were used when you launch instances using the AMI. The license configuration specifies how your licenses are counted (for example, by vCPUs, or the number of instances). See the&nbsp;<a href="https://docs.aws.amazon.com/license-manager/latest/userguide/create-license-configuration.html">Create a license configuration</a> and <a href="https://docs.aws.amazon.com/license-manager/latest/userguide/license-rules.html#ami-associations">Manually associating license configurations with AMI</a> documentation to learn how to create the license configuration and associate it with the AMI.</li> 
</ol>

<h2>Delegate your seller issued license management on AWS</h2> 
<p>As an issuer, from the delegated administrator account, create a license representing the entitlements of a seller-issued license, and grant it to the delegated administrator account. Once the delegated administrator accepts the grant, they can distribute the license to specific member accounts or all accounts in the organization. The delegated administrator account can view the license consumption from the License Manager Console. Note that if you’re granting to an organization,&nbsp;then the grants to member accounts will be auto-accepted. Refer to the&nbsp;<a href="https://aws.amazon.com/blogs/mt/ensure-license-compliance-aws-isvs-using-isv-seller-issued-licenses/">Ensure license compliance in AWS for ISVs using ISV seller-issued licenses</a> AWS post to learn how to implement license tracking and consumption of Independent Software Vendors (ISVs) issued licenses.</p> 
<h2>Use IAM and designate sub-administrators to licenses in the delegated administrator account</h2> 
<p>In the earlier section, you learned how to create a license configuration and share it across your organization from the delegated administrator account. In this section, you’ll learn how to set up IAM-based permissions to restrict a license administrator to view specific licenses. You can either choose to perform central license administration tasks in the organization from the delegated administrator account, or set up an IAM user in the delegated administrator account with sufficient permissions for specific license configuration(s) among other IAM permissions. Likewise, you can choose to offload license tracking (say, for multiple vendors) to multiple users, as well as think of them like sub-administrators.</p> 
<p>In this final section of the post, you’ll learn how to do the following:</p> 
<ol> 
 <li>From the <a href="https://us-east-1.console.aws.amazon.com/iamv2/home#/home">IAM console</a> of the delegated administrator account, create an IAM user with no assigned permissions. Refer to the <a href="https://docs.aws.amazon.com/IAM/latest/UserGuide/id_users_create.html#id_users_create_console">Creating IAM users (console)</a> documentation to learn how to create an IAM user from the console.</li> 
 <li>From the IAM console of the delegated administrator account, create a custom policy with the visual editor, as shown in the following figure. Note that the actions in the policy allow sufficient permissions only for the respective license and the license configuration. You can customize what actions to be added per your description of the policy for the IAM user.</li> 
</ol>

<ol start="3"> 
 <li>After you create the custom policy, you can attach it to the user created in Step 1. For more information, see <a href="https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_manage-attach-detach.html">Adding and removing IAM identity permissions</a>.</li> 
</ol> 
<p style="padding-left: 40px;">You can use <code>https://[AWS-account-ID or alias].</code> <a href="http://signin.aws.amazon.com/console">signin.aws.amazon.com/console</a> URL to sign in to the account sign-in page by substituting the correct AWS account ID number or account alias. This user in the delegated administrator account will now have sufficient permissions only for the specified license and the license configuration and your organization’s delegated administrator can offload license tracking activities to them.</p> 
<h2>Cleanup</h2> 
<p>Enabling License Manager cross-account discovery mode a host of resources, such as&nbsp;Amazon S3 buckets, AWS RAM Resource shares, <a href="https://aws.amazon.com/glue">AWS Glue</a> jobs, and <a href="http://aws.amazon.com/athena">Amazon Athena</a>. Refer to&nbsp;<a href="https://docs.aws.amazon.com/AmazonS3/latest/userguide/delete-bucket.html">Deleting a bucket</a>,&nbsp;<a href="https://docs.aws.amazon.com/ram/latest/userguide/working-with-sharing-delete.html">Deleting a resource share in AWS RAM</a>,&nbsp;<a href="https://docs.aws.amazon.com/glue/latest/ug/managing-jobs-chapter.html#delete-jobs">Delete jobs</a>,&nbsp;and&nbsp;<a href="https://docs.aws.amazon.com/glue/latest/dg/console-databases.html">Working with Databases on the AWS Glue Console</a> to learn how to clean up the aforementioned respective resources.&nbsp;Additionally, you may call the<a href="https://awscli.amazonaws.com/v2/documentation/api/latest/reference/organizations/deregister-delegated-administrator.html"> deregister-delegated-administrator</a>&nbsp;AWS CLI command to deregister a delegated administrator.</p> 
<h2>Conclusion</h2> 
<p>In this post, you learned how to set up&nbsp;delegated administrator and perform central license administration tasks in the organization, such as setting up cross-account discovery, sharing license configuration with other member accounts, and managing licenses.&nbsp;Delegated administrator is offered at no additional charge and is available in all AWS Regions where AWS Organizations is supported. License Manager supports one delegated administrator account at any time. I encourage you to use what you have learned here and consider using the delegated administrator feature in your own organization.&nbsp;If you have questions or feedback, then open an <a href="http://aws.amazon.com/support">AWS Support Case</a>.</p>
